# Contributing

All contributions, bug reports, bug fixes, documentation improvements, enhancements, and ideas are welcome.

## How to contribute

1. **Fork** this repository and clone your fork locally.
2. Create a **branch** for your changes (e.g. `bug-fix-typo`, `feature/calculator`).
3. Make your changes and **commit** with clear messages.
4. **Push** to your fork and open a **Pull Request** toward the original repository (`ibm-developer-skills-network/mcino-Introduction-to-Git-and-GitHub`).
5. Describe your changes in the PR and wait for review.

## Reporting bugs

Use the [Issues](https://github.com/ibm-developer-skills-network/mcino-Introduction-to-Git-and-GitHub/issues) section. Include a clear description, steps to reproduce, and your environment if relevant.

## Code of conduct

Please be respectful and constructive. See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) for details.
